# Fast Food Kitchen Model

## Описание

Данный проект представляет собой учебное программное обеспечение, разработанное в рамках курсового проекта по объектно-ориентированному программированию.  
Программа моделирует процесс работы кухни ресторана быстрого питания, включая формирование заказа, его последовательную обработку кухонными агрегатами и отображение результата пользователю.

Проект наглядно демонстрирует применение принципов ООП: наследование, инкапсуляция, полиморфизм и абстракция, а также показывает их практическую ценность при моделировании реальных процессов.

### Технологический стек
- Язык программирования: **Java**
- Графический интерфейс: **JavaFX**
- Сборка проекта: **Maven**
- Тип приложения: **Standalone desktop application**

### Статус проекта
Учебный проект (Stable)

---

## Архитектура

Архитектура проекта построена по принципу разделения ответственности между логикой пользовательского интерфейса и предметной областью.

- `HelloApplication` — точка входа в приложение
- `HelloController` — контроллер пользовательского интерфейса
- `TProcessingUnit` — абстрактный базовый класс кухонных агрегатов
- `TGrill`, `TFryer`, `TOven`, `TDrinkMachine` — конкретные кухонные агрегаты
- `TAssemblingTable` — агрегат финальной сборки заказа
- `TOrder` — модель заказа

## Проверка ПО

Проверка корректности работы программы осуществляется вручную:

- Пользователь выбирает необходимые элементы заказа с помощью флажков.
- Нажимает кнопку «Создать заказ».
- Программа формирует заказ и последовательно обрабатывает его кухонными агрегатами.
- В текстовой области отображается лог обработки и общее время выполнения заказа.
- Результаты обработки соответствуют выбранной конфигурации заказа.

## Проблемы

Известные ограничения текущей версии:

- отсутствует параллельная обработка заказов;
- отсутствует система приоритетов заказов;
- нет сохранения истории заказов между запусками программы.

Указанные ограничения не влияют на учебную ценность проекта.
